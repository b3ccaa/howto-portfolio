Customize the Portfolio

This responsive portfolio is ready to use and can easily be customized. Here, you find all what you need to create your own portfolio.

Writing Webpages

The given setup to create webpages supports you maintaining your portfolio in an easy way.
In the folder structure mentioned above you find in ../templates two created HTML templates. These templates include several "partials" which can be found in the folder with the same name. Partials are sections and areas of a webpage, such as the navigation and footer.

The advantage of this setup is the better maintainability and re-use of elements of the webpages. Imagine you want to change the navigation (on all pages). You only need to change one file and all HTML files will be generated automatically with the new navigation.

Before you create new pages and content you need to run the command grunt watch from the directory "howto_portfolio". Grunt notices if new files have been created or their content has been changed and then runs its task to create full HTML pages. This and other tasks are defined in Gruntfile.js. The generated .html files can be found in the folder ../dist.

Keep in mind that you name the page files (in the pages folder) pagename.dot.html. If you forget the .dot, Grunt can't run the task to compile the HTML files.

Writing CSS

CSS styles your website. Just as we want to have better maintainability for the HTML, we also like to easily change CSS by keeping an overview. That's why we use Less, which enables also more than simple styles. You can for instance define your colors in a variable (variable.less), or write functions (mixins.less).

All .less files will be translated to a single .css file (style.css).
You find all of the .less files in the ../less folder. There you can see that the styles have been structured into components (e.g. nav.less)

Before you make changes in the .less files, you need to setup your Less compiler (e.g. WinLess). The target folder needs to be ../dist/css.

Icon Fonts

Icon fonts help you creating re-usable and scalable icons without using images. You can even adjust the size and color by applying CSS without reducing the quality of the icon because it is a vector graphic.

In the folder structure you can find a folder called "icomoon-font". In there is a small set of icons that had been created for this responsible portfolio. If you like to add more icons, go to Icomoon and upload the file selection.json by clicking on "Import Icons".
To use the icon-font, copy the font files to ../dist/css/fonts, and copy and paste the content of the style.css in to icon-fonts.css which can be found in ../dist/css.
